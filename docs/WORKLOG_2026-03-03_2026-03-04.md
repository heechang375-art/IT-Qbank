# IT-Qbank 작업 정리 (2026-03-03 ~ 2026-03-04)

## 1) 공통 배경
- 초기 증상: 카테고리 로드 실패(HTTP 500/503), 결과 페이지 흰 화면, 한글 깨짐, AI 출제 실패(403/404/502), 문제 수 미충족(10 선택 시 7~8개), K8s 배포/라우팅 불안정.
- 목표: DB + AI 혼합 출제 안정화, 한글/시간/이력 저장 정상화, Docker/K8s 배포 가능한 상태로 정리.

## 2) 백엔드(backend/app.py) 수정 요약
1. AI 출제 경로 안정화
- `source=ai` 요청에서 AI 응답 실패 시 즉시 502 반환하던 흐름을 완화.
- AI 재시도/중복 제거/폴백 로직 보강.
- JSON 파싱 실패(따옴표/꼬리 쉼표/코드블록) 대응 강화.

2. 문제 수 미충족 이슈 대응
- 요청 개수(limit) 대비 반환 개수 부족 시 채움 로직 강화.
- AI 실패/중복 상황에서 대체 문제 생성 로직 추가.
- 변형 suffix가 정규화에 의해 중복 처리되던 부분 보정.

3. DB/스키마 예외 처리
- `created_at` 컬럼 중복 추가 시(1060) 예외 처리.
- SQLite 폴백 환경에서 멀티 워커 초기화 경합(`table already exists`) 완화.

4. 인코딩/한글 처리
- 한글 판별 정규식 및 카테고리 라벨 인코딩 깨짐 복구.
- 일부 깨진 문자열을 안전한 유니코드 이스케이프 형태로 정리.

5. 사용자 이력 저장
- 사용자 기준(`user_name`)으로 제출 결과 저장/조회 API 흐름 유지.
- 결과/이력 페이지에서 시도 내역 확인 가능 구조 유지.

6. 성능 상한 조정(응답 시간)
- AI 호출 시간 상한/요청 예산(환경변수) 조정:
  - `GEMINI_TIMEOUT`
  - `AI_REQUEST_BUDGET_SEC`
- 모델 후보 탐색 폭 축소로 지연 완화.

## 3) 프론트엔드(frontend/app.py) 수정 요약
1. 프록시 타임아웃 조정
- `FRONTEND_PROXY_TIMEOUT` 기본값을 60초로 조정해 무한 대기/긴 정체 완화.

2. API 프록시 경로 유지
- `/api/*` 요청을 백엔드로 프록시하는 구조 유지.
- 백엔드 연결 실패/타임아웃 에러 메시지 반환 동작 유지.

## 4) DB/문자셋/시간 관련 정리
1. 한글 저장 깨짐 대응
- MySQL 연결 시 `utf8mb4` 사용.
- DB/테이블 문자셋 점검을 전제로 한 수정 반복 진행.

2. 시간 처리
- 앱 내부 기준 시간을 KST(UTC+9) 기반으로 저장하도록 유지.
- DB/컨테이너 TZ 설정(Asia/Seoul) 반영 구조 유지.

## 5) Docker 관련 수정/정리
1. 이미지/실행 구조
- `frontend`, `backend` 이미지 재빌드/재배포 반복.
- 로컬에서 `docker compose` 기준 기능 점검(문제 조회/제출/이력).

2. AI 실패 시 동작
- Groq 실패 구간 이후 Gemini 기반으로 전환해 출제 흐름 사용.
- 키/모델/타임아웃 환경변수 정리.

## 6) Kubernetes(k8s) 수정 요약
1. 매니페스트 구조 정리
- 네임스페이스 파일, ordered apply 스크립트, gateway/network policy/rq 관련 파일 추가/정리.
- Ingress 기반에서 Gateway API 기반 라우팅으로 전환한 상태 정리.

2. 서비스 라우팅
- 현재 `frontend-service`는 `ClusterIP` 유지.
- 외부는 Gateway API(HTTPRoute) 경유 접속 전제.

3. 리소스/스케줄링
- backend replicas/리소스 요청값 조정 이력 반영.
- 노드 리소스 부족/스케줄 실패(Insufficient cpu/memory) 대응 위해 request/limit 조정 반복.

4. 보안/접근
- backend-only DB 접근 방향의 네트워크 정책 구성 정리.
- Secret/ConfigMap 분리 구성 정리.

## 7) 실제 장애 원인으로 확인된 주요 항목
1. HTTP 500/503
- DB 초기화 실패, 스키마 충돌, 앱 부팅 중 예외.

2. HTTP 502/504
- AI 생성 지연/실패 + 문제 수 채움 실패 조합.
- 프론트 프록시 타임아웃과 백엔드 지연의 합성 문제.

3. 한글 깨짐
- 소스 파일 인코딩 깨짐/정규식 깨짐/DB client 출력 환경 영향이 혼재.

4. K8s 접속 불가/오작동
- 공유 Gateway 환경, DNS 미전파, 라우트/서비스 경로 불일치.

## 8) 환경변수 변경 포인트(최종 반영 기준)
- `.env`, `.env.example`, `k8s/configmap.yaml`에 타임아웃/AI 예산 관련 키 반영.
- 핵심 키:
  - `GEMINI_TIMEOUT`
  - `AI_REQUEST_BUDGET_SEC`
  - `FRONTEND_PROXY_TIMEOUT`
  - `GEMINI_MODEL`
  - `GEMINI_API_KEY`

## 9) 현재 상태 요약

- Gateway API + ClusterIP 조합 구조로 정리됨.
- 단, 실제 클러스터 반영 결과는 이미지 태그/적용 시점/외부 DNS 상태에 따라 달라질 수 있음.
